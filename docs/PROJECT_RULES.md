# 🛑 PROJECT RULES (불변의 강제 규칙)

이 문서는 프로젝트의 **유일한 규칙 원본(Single Source of Truth)**입니다.
모든 작업자는 작업 시작 전 이 문서를 반드시 숙지해야 하며, 위반 시 세션이 즉시 종료될 수 있습니다.

---

## ⚠️ Zero Tolerance Policy (무관용 원칙)
> **"효율성보다 규정이 우선이다."**

### 1. 🔒 승인 우선 원칙 (Authorization First)
*   **규칙**: `git commit`, `git push` 명령은 사용자의 명시적인 승인(예: "진행해", "커밋해") 없이는 절대 실행할 수 없다.
*   **절차**:
    1.  코드 수정 및 검증 완료.
    2.  `rule_guard.ps1` 실행하여 문서 현행화 검증.
    3.  원인과 조치 설명 후 "커밋할까요?" 요청.
    4.  승인 획득 후 실행.

### 2. 📚 문서 무결성 원칙 (Documentation Integrity)
*   **규칙**: 코드 변경 시 다음 **5대 필수 문서**를 반드시 현행화해야 한다.
    1.  **`docs/PROJECT_RULES.md`** (본 문서): 규칙 변경 시 최우선 업데이트.
    2.  **`task.md`**: 작업 체크리스트 (최신순 정렬).
    3.  **`history.md`**: 작업 날짜와 상세 내역 기록.
    4.  **`README.md`**: 사용자 매뉴얼 및 기능 소개.
    5.  **`future_roadmap.md`**: 방향성 및 마일스톤 (또는 `docs/SERVER_SYNC_STRATEGY.md`).

### 3. 🇰🇷 한국어 소통 원칙 (Korean Only)
*   **규칙**: 모든 설명, 문서(Artifacts), 요약, 커밋 메시지(Feature 제외 본문), 워크스루는 반드시 **한국어**로 작성한다.

### 4. 🚫 로컬 빌드 금지 (No Local Build)
*   **규칙**: 로컬 환경에서 `flutter build`나 `flutter run`으로 배포 파일을 생성하지 않는다.
*   **대안**: 오직 `git push`를 통한 GitHub Actions CI/CD 빌드 결과만 신뢰한다.

### 5. 🌍 국제화 강제 (Mandatory L10n)
*   **규칙**: 텍스트 추가/수정 시 80개국 언어 ARB 파일을 반드시 동기화한다. (영어 Pivot 기준)

### 6. 🛡️ 코드 무결성 확인 (Code Integrity)
*   **규칙**: 코드 수정 후 반드시 **중복 코드 삽입 여부**와 **괄호(`{`, `}`) 매칭**을 재확인한다.
*   **배경**: 이전 작업에서 중복 코드로 인한 빌드 에러 빈번 발생. (Rule 8)

### 7. 💡 도움말 및 튜토리얼 일치 (Help & Tutorial)
*   **규칙**: UI 변경 시 도움말(JSON)과 튜토리얼(Highlights)의 타겟(`GlobalKey`)이 실제와 일치하는지 검증한다.
