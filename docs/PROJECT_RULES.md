# 🛑 PROJECT RULES (절대 규칙)

이 프로젝트는 다음 3가지 핵심 규칙을 **예외 없이** 따릅니다.

## 1. 한국어 소통 원칙 (Korean Only)
*   **모든 설명, 대화, 문서(Implementation Plan, Commit Message 등 포함)는 반드시 '한글'로 작성합니다.**
*   영어로 작성된 내용은 즉시 폐기하고 한글로 다시 작성해야 합니다.

## 2. 승인 우선 원칙 (Approval First)
*   **단계별 승인 필수**:
    *   계획(Plan) 수립 후 → **사용자 승인** → 코딩(Code) 진행.
    *   코딩(Code) 완료 후 → **사용자 승인** → 다음 작업 or 커밋(Commit) 진행.
*   사용자의 명시적인 "진행해", "좋아" 등의 승인 없이는 절대 다음 단계로 넘어가지 않습니다.

## 3. 포괄적 동기화 원칙 (Comprehensive Update)
*   **하나의 코딩 작업이 끝나면, 그로 인해 영향을 받는:**
    1.  **모든 프로젝트 문서** (`task.md`, `history.md`, `README.md` 등)를 즉시 현행화합니다.
    2.  **국제화(L10n)**가 필요한 경우, **지원하는 전체 언어(현재 80개국)**에 대해 빠짐없이 적용합니다.
*   문서나 언어 파일이 하나라도 누락된 상태에서의 커밋은 허용되지 않습니다.
