# Talkie 수익 모델 및 경제성 분석 보고서

## 1. 개요 (Overview)
본 문서는 Talkie 앱의 지속 가능한 서비스를 위한 과금 정책, 서버 유지 비용, 그리고 광고 수익성을 기술적으로 분석한 보고서입니다.

## 2. 과금 정책 (Pricing Policy)

| 구분 | 정책 내용 | 기술적 근거 |
| :--- | :--- | :--- |
| **번역 (Mode 1)** | **일일 20회 무료** | `UsageService`를 통한 하루 사용량 Strict Limit 적용. |
| **말하기 연습 (Mode 3)** | **무제한 무료** | `On-Device` STT/TTS(기기 내장 엔진) 사용으로 서버 비용 $0. |
| **가져오기 기능** | **무제한 무료** | 로컬 SQLite DB 직접 저장 방식을 사용하여 AI API 호출 비용 제거. |
| **리필 (Refill)** | **보상형 광고** | 광고 시청 시 번역 횟수 충전. (수익 창출 구간) |

## 3. 비용 시뮬레이션 (Cost Analysis)
**가정:** 사용자 10,000명 (DAU), 전원 일일 한도(20회) 모두 소진 시.

*   **API 모델:** Gemini 2.0 Flash (Tier 1 Pay-as-you-go)
*   **일일 요청 수:** 200,000회 (10,000명 x 20회)
*   **예상 토큰:** (입력 400 + 출력 50) 토큰 x 20만 회 = 9,000만 토큰
*   **일일 비용:** 약 **$9.00** (약 12,000원)
*   **월간 비용:** 약 **$270.00** (약 360,000원)

## 4. 수익성 분석 (Revenue Analysis)
보상형 광고(Rewarded Video)의 높은 참여율과 단가를 적용한 결과입니다.

*   **광고 참여율 가정:** 30% (사용자가 리필을 위해 자발적 시청)
*   **일일 시청 횟수:** 3,000회 (1만 명 중 3천 명)
*   **eCPM (1,000회 노출당 수익):** $25 (보수적 기준, 한국/미국 등 주요 국가)
*   **일일 예상 수익:** (3,000 / 1,000) * $25 = **$75.00**

### 💰 일일 손익 계산
*   수익 ($75) - 비용 ($9) = **+$66 (약 88,000원 흑자/일)**
*   **월간 순이익:** 약 **$1,980 (약 260만 원)**

## 5. 결론 및 리스크 평가 (Risk Assessment)
*   **서버 리스크:** **없음.** 백그라운드 동기화 및 로컬 우선 정책으로 서버 부하를 최소화함.
*   **비용 리스크:** **매우 낮음.** 무료 사용자(체리피커)가 많더라도, 소수의 광고 참여자가 전체 서버 비용을 상쇄하고 남는 "고수익 구조".
*   **확장성:** 사용자 수가 10만 명, 100만 명으로 늘어나도 수익 비율(Margin)은 동일하게 유지되거나 규모의 경제로 더 개선됨.

이 모델은 **초기 투자 비용 없이** 안전하게 서비스를 확장할 수 있는 가장 이상적인 구조입니다.
